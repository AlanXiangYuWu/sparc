# RMHA项目原理详解

## 一、问题定义

### 1.1 多机器人路径规划（MRPP）问题

**场景描述：**
- 网格世界：N×N 的二维网格（如 10×10, 40×40）
- 多个机器人：每个机器人是一个网格点，有起始位置和目标位置
- 障碍物：随机分布的障碍物（密度 0-50%）
- 目标：所有机器人无碰撞地到达各自目标

**动作空间：**
- 5个离散动作：上、下、左、右、停留

**观测空间：**
- 图像观测（8通道，3×3视野）：
  - 通道0-3：启发式地图（四个方向的移动建议）
  - 通道4：障碍物地图
  - 通道5：其他机器人位置
  - 通道6：自己的目标位置
  - 通道7：其他目标位置
- 向量观测（7维）：
  - dx, dy, d：到目标的归一化距离
  - re_t-1：前一时刻外在奖励
  - ri_t-1：前一时刻内在奖励
  - dmin_t-1：与历史位置的最小距离
  - a_t-1：前一动作

### 1.2 为什么困难？

1. **部分可观测**：每个机器人只能看到周围 3×3 的局部视野
2. **多智能体协作**：需要协调避免碰撞
3. **动态环境**：其他机器人也在移动
4. **规模扩展**：从训练时的8个扩展到测试时的128+个

---

## 二、为什么需要通信？

### 2.1 没有通信的问题

**场景示例：**
```
两个机器人在狭窄通道相遇：
R1 → → → → → R2
```

没有通信时：
- R1 不知道 R2 要过来
- R2 不知道 R1 要过来
- 结果：死锁或碰撞

### 2.2 通信的作用

通过通信，机器人可以：
- **分享位置信息**：告诉其他机器人"我在哪里"
- **分享计划**：告诉其他机器人"我要去哪里"
- **协调路径**：避免冲突，优化整体路径

---

## 三、传统方法的局限性

### 3.1 方法1：无通信（MAPPO baseline）

**特点：**
- 每个机器人独立决策
- 只基于局部观测

**问题：**
- 无法协调，容易碰撞
- 在复杂环境下成功率低（30%障碍物时只有40%成功率）

### 3.2 方法2：图神经网络（GNN）通信

**特点：**
- 使用图神经网络聚合邻居信息
- 所有邻居权重相同

**问题：**
- 忽略了距离的重要性
- 远处的机器人和近处的机器人权重相同，不合理

### 3.3 方法3：固定权重通信

**特点：**
- 距离越近，权重越大
- 权重是固定的（如：1/distance）

**问题：**
- 权重固定，无法学习最优策略
- 不能根据内容动态调整

---

## 四、RMHA的核心思想

### 4.1 核心创新

**RMHA = 空间关系增强的多头注意力**

核心思想：
> 让机器人根据**空间关系（距离）**和**通信内容**，**动态地**决定应该关注哪些其他机器人的信息。

### 4.2 直观理解

想象你在人群中找朋友：

**传统方法：**
- 只看距离：近的人更重要
- 问题：忽略了对方在说什么

**RMHA方法：**
- 同时考虑距离和内容
- 距离近 + 内容重要 = 高权重
- 距离远但内容关键 = 也可能高权重

**例子：**
- 机器人A：距离你5格，说"我要向左走"（与你的路径冲突）
- 机器人B：距离你3格，说"我要向右走"（与你的路径不冲突）
- RMHA会动态决定：虽然B更近，但A的路径与你冲突，可能更关注A

### 4.3 技术优势

1. **动态权重**：根据距离和内容动态计算注意力权重
2. **多头注意力**：从多个角度理解信息（类似多个人从不同角度看问题）
3. **空间关系编码**：将距离信息编码为高维向量，融入注意力计算

## 五、技术架构详解

### 5.1 整体流程

```
观测 → 编码器 → 通信模块 → 策略网络 → 动作
         ↓           ↓
      CNN+LSTM    RMHA
```

### 5.2 步骤1：观测编码

**输入：**
- 图像观测：(8, 3, 3) - 8通道的3×3视野
- 向量观测：(7,) - 7维向量

**处理：**
1. **CNN编码**：提取图像特征
   - 7层卷积 + 池化
   - 输出：特征向量
2. **全连接层**：融合图像和向量特征
3. **LSTM**：处理时序信息（记住历史状态）

**输出：**
- 编码特征：(256,) - 每个机器人的特征向量

**代码位置：** `models/encoder.py`

### 5.3 步骤2：RMHA通信（核心）

**输入：**
- 消息矩阵：(N, 256) - N个机器人的特征向量
- 距离矩阵：(N, N) - 机器人间的曼哈顿距离

**处理过程：**

#### 5.3.1 距离嵌入
```python
# 将距离编码为高维向量
distance_emb = DistanceEmbedding(distance_matrix)
# 输出：(N, N, 64) - 每对机器人间的距离特征
```

#### 5.3.2 计算Query, Key, Value
```python
# 融合距离信息到消息中
enhanced_messages_q = messages + W1(distance_emb)  # Query增强
enhanced_messages_k = messages + W2(distance_emb)  # Key增强

Q = W_q(enhanced_messages_q)  # Query
K = W_k(enhanced_messages_k)  # Key
V = W_v(messages)              # Value（不包含距离）
```

**关键点：**
- Q和K融合了距离信息
- V只包含消息内容（不包含距离）

#### 5.3.3 多头注意力计算
```python
# 计算注意力分数
attn_scores = Q @ K^T / sqrt(head_dim)
# 输出：(num_heads, N, N) - 每个头对每对机器人的注意力分数

# Softmax归一化
attn_weights = softmax(attn_scores)
# 输出：(num_heads, N, N) - 注意力权重（和为1）

# 加权聚合
updated_messages = attn_weights @ V
# 输出：(N, 256) - 更新后的消息
```

**注意力权重的含义：**
- `attn_weights[i, j, k]` = 机器人j对机器人k的注意力权重
- 权重高 = 机器人j更关注机器人k的信息
- 权重由距离和内容共同决定

#### 5.3.4 多层处理
```python
# 通过3层RMHA（类似Transformer的3层）
for layer in rmha_layers:
    messages = layer(messages, distance_matrix)
```

**代码位置：** `models/rmha.py`

### 5.4 步骤3：策略网络

**输入：**
- 编码特征 + 通信消息：(256 + 256 = 512,)

**处理：**
1. 全连接层：512 → 256 → 128
2. 输出层：128 → 5（5个动作的概率分布）

**输出：**
- 动作概率：(5,) - 每个动作的概率
- 采样动作：根据概率采样一个动作

**代码位置：** `models/policy.py`

### 5.5 步骤4：价值网络

**输入：**
- 编码特征 + 通信消息：(512,)

**输出：**
- 外在价值：预测未来外在奖励的累积
- 内在价值：预测未来内在奖励的累积

**用途：**
- 用于PPO训练（计算优势函数）

**代码位置：** `models/value.py`

## 六、训练过程（MAPPO）

### 6.1 MAPPO是什么？

**MAPPO = Multi-Agent PPO**

- **PPO**：Proximal Policy Optimization（近端策略优化）
- **Multi-Agent**：多智能体版本

**核心思想：**
- 集中式训练：训练时所有机器人共享参数
- 分散式执行：执行时每个机器人独立决策

### 6.2 训练流程

#### 6.2.1 收集经验
```python
# 运行一个episode（最多256步）
for step in range(256):
    # 1. 获取观测
    obs = env.get_observations()
    
    # 2. 智能体选择动作
    actions = agent.forward(obs, deterministic=False)
    
    # 3. 环境执行动作
    next_obs, rewards, done, info = env.step(actions)
    
    # 4. 存储经验
    buffer.add(obs, actions, rewards, ...)
```

#### 6.2.2 计算回报和优势
```python
# 使用GAE（Generalized Advantage Estimation）计算优势
advantages = compute_gae(
    rewards, values, gamma=0.99, gae_lambda=0.95
)
```

#### 6.2.3 PPO更新
```python
# 多轮更新（4轮）
for epoch in range(4):
    # 小批次采样
    for batch in batches:
        # 1. 评估动作
        new_log_probs = agent.evaluate_actions(batch)
        
        # 2. 计算比率
        ratio = exp(new_log_probs - old_log_probs)
        
        # 3. PPO裁剪目标
        policy_loss = -min(
            ratio * advantages,
            clip(ratio, 0.8, 1.2) * advantages
        )
        
        # 4. 价值损失
        value_loss = (values - returns)^2
        
        # 5. 总损失
        loss = policy_loss + value_loss + entropy_loss
        
        # 6. 反向传播
        loss.backward()
        optimizer.step()
```

**代码位置：** `algorithms/mappo.py`

### 6.3 奖励设计

**外在奖励：**
- 移动：-0.3（鼓励移动）
- 停留在目标：0.0
- 停留在非目标：-0.3
- 碰撞：-2.0（严重惩罚）
- 到达目标：+1.0（一次性奖励）

**内在奖励（探索激励）：**
- 访问新区域：+0.1
- 鼓励探索未访问的位置

## 七、RMHA vs 其他方法

### 7.1 对比实验

| 方法 | 0%障碍物 | 15%障碍物 | 30%障碍物 |
|------|----------|-----------|-----------|
| MAPPO（无通信） | ~95% | ~60% | ~40% |
| MAPPO+GNN | ~98% | ~70% | ~50% |
| **RMHA** | **~100%** | **~90%** | **~75%** |

### 7.2 为什么RMHA更好？

1. **空间关系编码**：融合距离信息，让注意力更合理
2. **动态权重**：根据内容动态调整，不是固定权重
3. **多头注意力**：从多个角度理解信息

1. **空间关系增强**：将距离信息融入注意力机制
2. **动态通信权重**：根据距离和内容动态调整
3. **可扩展性**：从8个机器人扩展到128+个

### 9.2 应用场景

- 仓库机器人路径规划
